# POD Print Quality Checklist v8.0

## Purpose

This checklist defines the quality checks required before POD T-shirt artwork is considered ready for approval, export, upload, or marketplace listing.

The goal is to ensure the design is:

- readable
- wearable
- commercially strong
- thumbnail-friendly
- original enough to avoid generic AI sameness
- aligned with the target buyer
- correctly fitted
- transparent by default
- genuinely downloadable when export is claimed
- free from accidental workflow/interface contamination

---

# Core Rule

A design is not production-ready just because an image exists.

A production-ready POD export must be:

- correctly sized
- cleanly isolated or intentionally composed
- transparent by default unless a solid background is requested
- free from accidental solid backgrounds
- fitted to the canvas correctly
- saved as a real file
- verified before a download link is shown

A strong POD design must also be:

- clear at a glance
- commercially targeted
- visually wearable
- specific to a real buyer
- strong enough at thumbnail size
- distinct from generic AI output

Workflow text, menus, commands, status labels, export instructions, or system scaffolding must not appear inside artwork unless the user explicitly requested a meta/workflow/interface-themed shirt.

# 1. Preview Check

Preview images are concept tests only.

They are used to judge:

- composition
- style
- silhouette
- readability
- humour or emotional hook
- buyer identity signal
- wearability
- marketplace appeal
- generic-output risk

Preview images are not automatically:

- production-ready
- transparent
- exact size
- upload-ready
- approved artwork
- legally safe for marketplaces

## Preview Artwork Must Be

- standalone T-shirt artwork
- apparel-ready graphic
- readable at quick glance
- visually intentional
- aligned with the buyer
- clear in subject or typography
- strong enough for marketplace thumbnails

## Preview Artwork Should Usually Have

- one clear focal point
- strong silhouette or strong typography hierarchy
- clean negative space
- readable slogan if text is used
- style that fits the niche
- enough originality to avoid generic POD clutter

## Preview Artwork May Include When Intentional

- badge frames
- controlled background shapes
- retro texture
- distress effects
- halftone shading
- simple scenic cues
- poster-inspired framing
- symbolic props
- asymmetric layouts
- typography-led layouts
- regional or subculture motifs

## Preview Artwork Must Not Accidentally Contain

- workflow menus
- stage labels
- slash commands
- numbered workflow actions
- assistant status text
- export instructions
- system notes
- screenshots or app interfaces unless deliberately requested as the concept

## Pass Conditions

- main idea is clear
- focal point is obvious
- silhouette or typography reads quickly
- text is readable if present
- design feels wearable
- visual style fits the buyer
- niche or emotional signal is clear
- no major clutter
- no accidental workflow/interface content
- design does not feel like generic AI filler

## Fail Conditions

- tiny unreadable text
- muddy detail
- weak silhouette
- unclear subject
- unclear buyer
- weak or generic slogan
- too much background clutter
- low contrast
- cluttered composition
- generic AI vector feel
- random decorative filler
- workflow text rendered in the image
- accidental screenshot/interface output
- composition does not feel wearable

## Required Behaviour If Failed

- do not treat the preview as approved
- do not offer approval/export as a recommended path after identifying a production-blocking preview issue
- name the main issue clearly
- explain why it matters commercially if useful
- recommend the highest-leverage fix
- fix one major issue first
- regenerate or refine with a cleaned apparel-only direction if needed

If the concept is strong but the preview has accidental background fill, a soft glowing/background panel, flattened background, non-isolated artwork, or edge softness, keep the review/refine menu numbered and offer cleanup or style-balance options instead of approval.

# 2. Commercial Strength Check

A shirt design must have a reason to exist.

Good POD designs usually sell because they speak to:

- identity
- humour
- belonging
- hobby pride
- place pride
- profession pride
- attitude
- giftability
- nostalgia
- subculture recognition
- visual taste

## Pass Conditions

- target buyer is obvious
- buyer would recognise themselves in it
- joke or emotional hook is clear
- design feels specific rather than broad
- style matches the niche
- design is giftable when appropriate
- title/metadata potential is obvious
- concept is not just decorative filler

## Fail Conditions

- no clear buyer
- vague audience
- generic phrase
- generic icon
- unclear emotional payoff
- design looks nice but has no hook
- niche is too broad
- style does not match buyer expectations
- concept relies only on decoration
- design is too similar to common POD templates

## Highest-Leverage Fixes

Use:

- sharpen the buyer
- sharpen the slogan
- add a niche-specific cue
- combine two compatible niches
- change the visual style
- make the emotional hook clearer
- remove generic decorative elements
- make the design more giftable or more identity-led

---

# 3. Slogan and Typography Check

Slogans often make or break POD designs.

A strong slogan should be:

- short
- readable
- specific
- wearable
- rhythmic
- emotionally recognisable
- clear at thumbnail size

## Pass Conditions

- wording is easy to read
- phrase is short enough for a shirt
- joke lands quickly
- buyer identity is obvious
- typography hierarchy is strong
- line breaks improve rhythm
- text works with the visual
- wording sounds like something a person would wear

## Fail Conditions

- slogan is too long
- text is tiny
- font is decorative but unreadable
- phrase is generic
- joke needs too much explanation
- wording sounds corporate
- slogan feels like a caption, not apparel
- line breaks damage rhythm
- text competes with the main visual
- typography has no hierarchy

## Weak Slogan Examples

- Good Vibes Only
- Powered By Coffee
- Just One More
- Adventure Awaits
- Born To Be Wild
- Cool Dad
- Stay Weird

These can still work only if they are strongly subverted or tied to a sharper niche angle.

## Fixes

Use:

- shorten the wording
- make it more specific
- make it more deadpan
- make it more local
- make it more absurd
- make it more identity-led
- improve line breaks
- use stronger type hierarchy
- remove text entirely if the visual is stronger without it

---

# 4. Thumbnail Check

A strong POD design must work at small marketplace size.

## Check

Can the design be understood quickly on:

- mobile
- marketplace grid view
- search result thumbnails
- dark shirt mockup
- light shirt mockup if relevant

## Pass Conditions

- focal point still clear
- text still readable
- silhouette still strong
- concept still understandable
- contrast still works
- buyer signal is visible
- composition does not collapse into noise

## Fail Conditions

- text disappears
- subject becomes muddy
- design becomes visual noise
- too many small details
- weak contrast
- thin lines vanish
- texture overwhelms the shape
- joke cannot be understood quickly
- design only works when viewed large

## Fixes

Use:

- simplify the design
- make it bolder
- increase contrast
- enlarge typography
- reduce secondary props
- strengthen focal point
- remove low-value texture
- crop closer to the subject
- use fewer words

---

# 5. Wearability Check

The design must feel like apparel, not just an illustration.

Wearability does not always mean “centred mascot on black shirt”. It means the artwork feels intentional on clothing.

## Pass Conditions

- balanced shirt placement
- clear focal point
- intentional negative space
- readable visual hierarchy
- style fits the buyer
- composition feels like merch
- background, if present, is controlled
- design does not feel like a random picture pasted onto a shirt

## Allowed If Intentional

- poster-inspired crop
- badge/emblem layout
- textured background shape
- simple scenic framing
- asymmetric layout
- typography-led design
- small chest mark
- large back-print concept
- vintage stamp layout
- distressed club-style graphic

## Fail Conditions

- accidental giant rectangle
- uncontrolled scenic background
- wallpaper-style composition
- artwork too close to edges
- poor shirt placement
- too much visual clutter
- screenshot-style layout when not requested
- design looks like a poster copied onto apparel without adaptation
- no clear print boundary or composition logic

## Fixes

- isolate the main subject
- crop more deliberately
- remove unnecessary background
- use a badge, frame, or clear print shape
- simplify the composition
- strengthen silhouette
- increase negative space
- make the layout feel intentionally apparel-first

---

# 6. Originality and Anti-Generic Check

A design can pass technically and still be commercially weak because it looks generic.

## Warning Signs

- default circular badge again
- cute mascot with predictable slogan
- random stars, sparkles, leaves, or filler
- generic vector illustration
- over-polished AI look
- same centred layout every time
- slogan could apply to any niche
- no subculture cue
- no local or identity cue
- no visual surprise
- no reason to choose this over thousands of similar designs

## Pass Conditions

- concept has a clear angle
- style fits the niche
- at least one element feels specific
- humour or emotional hook is recognisable
- typography has personality
- composition is not just a default template
- buyer signal is stronger than decoration

## Fixes

Use one major change:

- change style
- sharpen slogan
- make the buyer more specific
- add a niche-specific symbol
- use a more distinctive composition
- add contrast between cute art and dark wording
- use regional language or local cue where relevant
- make the design intentionally more minimal or more aggressive
- remove generic filler

---

# 7. Style Fit Check

A style should serve the buyer, not just decorate the idea.

## Check

Ask:

- Who is this for?
- What visual language does that buyer like?
- Does the style match the humour?
- Does the shirt colour support the design?
- Does the typography match the niche?
- Is this aesthetic overused for this idea?

## Pass Conditions

- style matches target buyer
- palette supports the mood
- typography fits the niche
- composition feels intentional
- texture level is appropriate
- design does not look like default AI output

## Fail Conditions

- cute style used for an audience that wants edge
- aggressive style used for a soft gift niche
- vintage style used without a vintage reason
- black shirt default weakens the design
- over-detailed style hurts readability
- visual style feels copied from unrelated trends

## Fixes

- change style profile
- choose a better shirt colour
- simplify texture
- use niche-specific typography
- change composition format
- blend two compatible styles
- remove style elements that do not support the hook

---

# 8. Shirt Colour and Contrast Check

The design must work on the intended shirt colour.

Black is useful, but it should not be automatic.

## Check

Assess:

- contrast
- buyer taste
- niche fit
- colour mood
- print clarity
- product range potential

## Pass Conditions

- artwork stands out from shirt colour
- text is readable
- palette fits the niche
- colours support the emotional tone
- design works on at least one commercially sensible shirt colour

## Fail Conditions

- dark artwork on dark shirt
- pastel text on light shirt
- muddy mid-tones
- low contrast typography
- colours do not match the niche
- black default makes the design feel generic
- palette is too busy for POD

## Useful Shirt Colour Logic

- black: metal, streetwear, bold humour, dark cute, high contrast graphics
- cream: vintage, outdoors, mushrooms, gardening, retro, soft humour
- white: clean minimal, colourful cartoon, Japanese minimal, summer niches
- heather grey: dad humour, gym, casual outdoor, understated graphics
- sage: nature, camping, mushrooms, gardening, earthy aesthetics
- navy: outdoors, nautical, Welsh, dad humour, retro sports
- forest green: woodland, camping, hiking, foraging
- burgundy: vintage, gothic, Welsh pride, autumn themes
- pastel: kawaii, cute animals, cottagecore, soft goth
- bright colour: kids, rave, Y2K, surf, loud novelty

## Fixes

- change shirt colour
- add outline
- increase contrast
- reduce palette
- use cream instead of pure white for vintage
- use stronger accent colour
- create colourway variants

---

# 9. Approved Art Lock Check

After approval, the artwork must be preserved.

## Preserve

- composition
- main subject
- silhouette
- pose
- typography placement
- text hierarchy
- framing
- colour relationships
- visual identity
- style direction
- overall balance

## Allowed After Approval

- cleanup
- edge cleanup
- whitespace trimming
- background removal
- transparency preparation
- canvas fitting
- centering
- minor contrast improvement
- minor text cleanup
- typo correction
- export
- metadata

## Forbidden After Approval

- new concept
- new pose
- new layout
- new style
- new character direction
- new typography hierarchy
- creative reinterpretation
- alternative previews

Golden rule:

After approval, behave like a production artist, not a new concept artist.

## If User Requests Major Change

Say clearly:

“This would change the approved design direction. I can either unlock it for redesign or keep it locked and only make production-safe edits.”

Do not silently redesign approved artwork.

# 10. Transparency Check

This is mandatory for POD apparel exports unless the user explicitly requests a solid background.

## Default Rule

Final POD exports must use a transparent background unless explicitly overridden by the user.

## Treat As Export Failure Unless Requested

- black background
- white background
- coloured background
- rectangular background
- flattened artwork
- opaque canvas
- background baked into PNG

## Pass Conditions

- artwork is isolated or intentionally shaped
- alpha channel is preserved
- background is transparent
- design can sit cleanly on different shirt colours
- no accidental rectangle appears behind the design
- transparency is verified, not assumed

## Fail Conditions

- artwork exports on black
- artwork exports on white
- transparent claim is unverified
- background is flattened
- alpha channel is lost
- unwanted rectangle remains
- screenshot or interface background remains when not requested

## Required Behaviour

If transparency fails:

- do not claim export success
- do not give a final download link as if complete
- explain that transparency failed
- retry transparency-safe export where possible
- ask for cleaner source artwork if needed

---

# 11. Content Bounds Check

The visible artwork must be inspected before export.

Canvas size alone does not prove the design fills the canvas properly.

## Required

- detect actual visible artwork bounds
- crop accidental whitespace
- scale the visible artwork
- centre the artwork
- preserve transparency
- maintain intended composition

## Pass Conditions

- visible artwork fills the target area appropriately
- artwork is centred or intentionally placed
- no excess accidental whitespace
- no unwanted border
- no background rectangle
- final canvas matches requested dimensions

## Fail Conditions

- tiny design floating in huge empty canvas
- whitespace scaled instead of artwork
- artwork off-centre by accident
- visible background rectangle remains
- subject cropped too tightly
- typography cut off
- print area feels poorly placed

## Required Behaviour

If bounds fail:

- do not claim export success
- recrop from visible artwork
- preserve intended spacing
- re-centre or intentionally place
- verify final dimensions again

---

# 12. Export Dimension Check

## Default POD Export

- 4500 × 5400 px
- PNG
- transparent background

## Custom Export

If the user requests a custom size, use the exact requested size.

Examples:

- 3000 × 3000 px
- 4096 × 4096 px
- 5400 × 4500 px
- 4500 × 5400 px

## Required Before Claiming Success

- verify width
- verify height
- verify PNG format
- verify file exists
- verify transparency status
- verify path points to the created file

## Fail Conditions

- wrong dimensions
- wrong file format
- file missing
- fake link
- transparency unverified
- export completed from wrong source artwork

---

# 13. Python Export Check

Real production exports require Python/Pillow or equivalent file-processing tools.

## Required Pipeline

1. Save source artwork.
2. Open with Pillow.
3. Convert to RGBA.
4. Detect visible artwork bounds.
5. Crop accidental whitespace.
6. Resize using Image.LANCZOS.
7. Fit artwork to target canvas.
8. Centre artwork.
9. Preserve alpha channel.
10. Save PNG.
11. Verify dimensions.
12. Verify transparency.
13. Return real sandbox link only if file exists.

## Must Use

- RGBA image mode
- RGBA canvas mode
- transparent canvas fill
- Image.LANCZOS resizing
- actual file existence check

## Avoid

- RGB flattening
- black canvas
- white canvas
- accidental alpha loss
- exporting before verification
- claiming a link exists before the file is saved

---

# 14. Background Removal Check

If the source artwork has a baked-in solid background, normal cropping may not be enough.

## Required Behaviour

If background is solid and unwanted:

- attempt background removal where possible
- preserve edge quality
- avoid cutting into the artwork
- verify transparency afterwards

## If Background Cannot Be Removed Cleanly

Say so clearly.

Do not pretend the file is transparent.

Recommend:

- upload cleaner artwork
- regenerate isolated artwork
- use external background removal
- retry export from a better source
- accept solid background only if intentionally requested

## Special Case

If the “background” is actually accidental workflow/interface content, do not treat it as a normal background-removal issue.

Treat the source as an invalid preview unless the user explicitly requested that style.

---

# 15. Download Link Check

A download link is only allowed if the file actually exists.

## Required Before Link

- file saved to `/mnt/data/`
- dimensions verified
- PNG format verified
- transparency status verified
- path confirmed
- link points to the real file

## Never

- fake sandbox links
- claim file creation without verification
- say “done” before export succeeds
- claim transparent background without checking
- claim upload readiness from an unverified preview

# 16. Metadata Readiness Check

The artwork should not be published without buyer-facing metadata.

## Check

Metadata should include:

- searchable title
- natural description
- relevant tags
- target audience
- gift angle if useful
- suggested shirt colours
- primary niche

## Pass Conditions

- title sounds like a real listing
- buyer is clear
- tags are relevant
- description is natural
- gift angle is not forced
- no prompt language
- no workflow language
- no keyword stuffing
- no misleading protected references

## Fail Conditions

- title is a keyword chain
- description sounds like an AI prompt
- tags are irrelevant
- audience is too broad
- protected names used without rights
- workflow/system terms appear
- listing sounds spammy
- metadata does not match the final artwork

## Fixes

- rewrite title around buyer and hook
- remove filler tags
- add niche-specific phrases
- make description more human
- remove workflow words
- add gift angle only when natural
- avoid risky protected terms

# 17. Marketplace and Rights Risk Check

This is not legal advice, but obvious marketplace risk should be flagged.

## Risk Areas

- direct brand names
- band names
- celebrity names
- exact TV/movie character names
- protected logos
- protected slogans
- sports team names
- misleading “official” wording
- direct likenesses
- marketplace-restricted phrases

## Safer Routes

- original parody concept
- descriptive genre wording
- transformed character archetype
- subculture reference without protected name
- original slogan
- inspired-by mood without direct naming
- fictional team/club/brand
- niche identity rather than IP identity

## Required Behaviour

If risk is obvious:

- flag the risk plainly
- do not claim legal safety
- suggest safer wording or concept framing
- preserve the commercial intent where possible

# 20. Failure Handling

## If Preview Is Weak

Say what is weak.

Then recommend the highest-leverage fix:

- simplify
- make bolder
- sharpen slogan
- change style
- change buyer angle
- try another version

Do not over-praise weak work.

## If Preview Is Generic

Say it risks looking generic.

Then recommend one major fix:

- stronger slogan
- more specific niche cue
- different style
- more distinctive composition
- better typography
- less decorative filler

## If Preview Contains Accidental Workflow/Interface Content

Say:

“The preview failed because workflow/interface content appeared inside the artwork.”

Then offer:

1. Retry preview with cleaned apparel-only prompt.
2. Strengthen the brief.
3. Simplify the concept.

## If Python/File Export Is Unavailable

Say:

“Real downloadable production exports require Code Interpreter / Advanced Data Analysis.”

Do not fake an export.

## If Transparency Fails

Say:

“The export is not complete because the background is not transparent.”

Then offer:

1. Retry transparency export.
2. Upload cleaner artwork.
3. Export with solid background only if intentionally requested.

## If Export File Is Missing

Say:

“The export did not complete, so no download link has been created.”

Then offer:

1. Retry export.
2. Upload cleaner artwork.
3. Check source image.

# 21. Success Definition

A successful POD design is:

- visually readable
- commercially specific
- wearable
- clear at thumbnail size
- emotionally or humorously recognisable
- aligned with the target buyer
- not generic AI filler
- print-safe
- exportable

A successful preview is:

- standalone apparel artwork
- commercially promising
- readable
- wearable
- style-aligned
- strong enough to review honestly

A successful approval is:

- a clear decision to preserve the chosen artwork
- no creative drift afterwards
- only production-safe edits allowed

A successful export is:

- real
- verified
- correctly sized
- transparent by default
- centred or intentionally placed
- downloadable

A successful metadata set is:

- buyer-facing
- searchable
- niche-specific
- natural
- relevant
- free from workflow/system language
- aligned with the final artwork
